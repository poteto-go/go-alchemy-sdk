# Erc1155

You can execute ERC1155 (multi-token) read and transfer methods from the specified wallet.
Each read method delegates to the [Erc1155 namespace](../erc1155-namespace/BalanceOf.md).

:::note

Since it performs calls via bytecode, it does not require the contract implementation and can be called as long as the address is available.

:::

:::warning

- It requires connected wallet.
- It does not work on non-Ethereum compatible networks.

:::

```go
func main() {
	alchemy = gas.NewAlchemy(setting)
	w, _ := wallet.New("<privateKey>")
	w.Connect(alchemy.GetProvider())

	// call each method
	w.ERC1155().MethodXXX()
}
```

## Read Methods

You can fetch ERC1155 balances and metadata:

- [BalanceOf](../erc1155-namespace/BalanceOf.md)
- [BalanceOfBatch](../erc1155-namespace/BalanceOfBatch.md)
- [Uri](../erc1155-namespace/Uri.md)

```go
balance, err := w.ERC1155().BalanceOfToken("<contractAddress>", "<account>", big.NewInt(1))
balances, err := w.ERC1155().BalanceOfBatch(
	"<contractAddress>",
	[]string{"<account1>", "<account2>"},
	[]*big.Int{big.NewInt(1), big.NewInt(2)},
)
uri, err := w.ERC1155().Uri("<contractAddress>", big.NewInt(1))
```

## Write Methods

You can transfer ERC1155 tokens held by (or approved to) the connected wallet.
Each method has a `NoWait` variant that returns the transaction hash without
waiting for the transaction to be mined. The trailing `gasLimit *uint64` is
optional — pass `nil` to use the default (`300000`).

### SafeTransferFrom & SafeTransferFromNoWait

Calls `safeTransferFrom(address,address,uint256,uint256,bytes)` — transfers `amount`
units of token `id` from `fromAddress` to `toAddress`. `data` is forwarded to the
recipient's `onERC1155Received` hook.

```go
data := []byte{}

receipt, err := w.ERC1155().SafeTransferFrom(
	context.Background(),
	"<contractAddress>",
	"<fromAddress>",
	"<toAddress>",
	big.NewInt(1),  // token id
	big.NewInt(5),  // amount
	data,
	nil, // use default gas limit (300000)
)

// or without waiting for the receipt
txHash, err := w.ERC1155().SafeTransferFromNoWait(
	"<contractAddress>",
	"<fromAddress>",
	"<toAddress>",
	big.NewInt(1),
	big.NewInt(5),
	data,
	nil,
)
```

### SafeBatchTransferFrom & SafeBatchTransferFromNoWait

Calls `safeBatchTransferFrom(address,address,uint256[],uint256[],bytes)` — transfers
multiple token types in a single transaction. `ids` and `amounts` must have the same
length.

```go
ids := []*big.Int{big.NewInt(1), big.NewInt(2)}
amounts := []*big.Int{big.NewInt(5), big.NewInt(10)}
data := []byte{}

receipt, err := w.ERC1155().SafeBatchTransferFrom(
	context.Background(),
	"<contractAddress>",
	"<fromAddress>",
	"<toAddress>",
	ids,
	amounts,
	data,
	nil,
)

// or without waiting for the receipt
txHash, err := w.ERC1155().SafeBatchTransferFromNoWait(
	"<contractAddress>",
	"<fromAddress>",
	"<toAddress>",
	ids,
	amounts,
	data,
	nil,
)
```

### SetApprovalForAll & SetApprovalForAllNoWait

Grant or revoke an operator's approval to manage **all** of the connected
wallet's ERC1155 tokens in the collection. Pass `approved=true` to grant and
`false` to revoke.

```go
receipt, err := w.ERC1155().SetApprovalForAll(
	context.Background(),
	"<contractAddress>",
	"<operatorAddress>",
	true,
	nil,
)

txHash, err := w.ERC1155().SetApprovalForAllNoWait("<contractAddress>", "<operatorAddress>", true, nil)
```
